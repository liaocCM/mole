## 21. [portless](https://github.com/vercel-labs/portless)
> Replace port numbers with stable, named .localhost URLs. For humans and agents.

**Language:** TypeScript  |  **Stars:** 3,681  |  **Forks:** 122  |  **Score:** 3,100  |  **Created:** 2026-02-15

**Background:** Portless from Vercel Labs replaces arbitrary localhost port numbers with stable, named .localhost URLs. Running `portless run next dev` serves the app at `http://myapp.localhost:1355` instead of the default `http://localhost:3000`. A proxy auto-starts to route named URLs to the correct ports.

**Problem it solves:** Local development with port numbers is fragile: port conflicts, memorizing which service runs on which port, and hardcoded port values in configs that break when ports change. Portless assigns stable, memorable names to local services, making the development experience more ergonomic for both human developers and AI agents that need to reference running services.

**Why another one?** The "for humans and agents" tagline captures the dual audience. AI coding agents frequently need to interact with locally running services and struggle with dynamic port assignments. Named URLs provide stable, predictable addresses that agents can reference without parsing startup logs for port numbers. Vercel Labs backing adds distribution and credibility.
